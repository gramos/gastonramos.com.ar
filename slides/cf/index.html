Why Cloud Foundry matters to hackers?

  • It is open source. You can modify it. In fact we are doing it.

  • Has no vendor locking.

Cloud Foundry providers: AppFog, cloudfoundry.com, pass.io

Cloud Foundry Architectural Goals

  • No single point of failure.
  • Distributed state.
  • Self healing.
  • Horizontally scalable.

We will attend only to the main components of cf, which are the router, the cloud_controller, Dea and Nats.

CF Core Components

( Almost all components written in ruby )

Nats

  • All cloud foundry components communicate through nats.

  • publish-subscribe messaging system.

Cloud Controller

  • REST API endpoints for clients

  • Maintains a database with tables for user information, like organizations and spaces.

Droplet Execution Agent (DEA)

  • Takes care of managing an application instance's lifecycle.

  • It can be instructed by the CC to start and stop application instances.

Droplet Execution Agent (DEA)

  • It keeps track of all started app instances

  • Periodically broadcasts messages about their state over NATS.
