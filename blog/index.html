Building rubinius debian package

Wed Aug 06 20:13:51 -0300 2008

I’m trying to build a debian package for rubinius, my knowledges about vm and c compiling are very litle, but… here we go!

I’m going to see required libs to run rubinius


[gramos@marlasina] cd ~/src/rubinius/

[gramos@marlasina] ldd shotgun/lib/librubinius-0.9.0.so

 linux-gate.so.1 =>  (0xffffe000)
 libz.so.1 => /usr/lib/libz.so.1 (0xb7dfa000)
 libm.so.6 => /lib/libm.so.6 (0xb7dd5000)
 libdl.so.2 => /lib/libdl.so.2 (0xb7dd1000)
 librt.so.1 => /lib/librt.so.1 (0xb7dc7000)
 libcrypt.so.1 => /lib/libcrypt.so.1 (0xb7d95000)
 libc.so.6 => /lib/libc.so.6 (0xb7c60000)
 /lib/ld-linux.so.2 (0x80000000)
 libpthread.so.0 => /lib/libpthread.so.0 (0xb7c49000)

[gramos@marlasina] dlocate /usr/lib/libz.so.1

  zlib1g: /usr/lib/libz.so.1.2.3.3
  zlib1g: /usr/lib/libz.so.1

As we can see, the package’s name is zlib1g


[gramos@marlasina] dpkg -l | grep zlib1g

ii  zlib1g       1:1.2.3.3.dfsg-12  compression library - runtime
ii  zlib1g-dev   1:1.2.3.3.dfsg-12  compression library - development

[gramos@marlasina] ldd /usr/lib/libz.so.1
   linux-gate.so.1 =>  (0xffffe000)
   libc.so.6 => /lib/libc.so.6 (0xb7df6000)
   /lib/ld-linux.so.2 (0x80000000)

[gramos@marlasina] dlocate /lib/libc.so.6
  libc6: /lib/libc.so.6

[gramos@marlasina] dlocate /lib/libdl.so.2
  libc6: /lib/libdl.so.2

[gramos@marlasina] dlocate /lib/librt.so.1
  libc6: /lib/librt.so.1

[gramos@marlasina] dlocate /lib/libcrypt.so.1
  libc6: /lib/libcrypt.so.1

[gramos@marlasina] dlocate /lib/libpthread.so.0
  libc6: /lib/libpthread.so.0

[gramos@marlasina] dpkg -l | grep libc6

ii  libc6      2.7-10  GNU C Library: Shared libraries
ii  libc6-dev  2.7-10  GNU C Library: Development Libraries and Headers

As we can see, the package’s name is libc6, in theory only two libs are required: libc6 (>= 2.4) and zlib1g.

apt-get install libc6 zlib1g
